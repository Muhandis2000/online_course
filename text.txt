Значение файла main.go
Файл main.go — это точка входа в приложение на Go. Он отвечает за запуск программы и координацию всех её компонентов. В контексте проекта онлайн-школы этот файл выполняет следующие ключевые задачи:

Загрузка конфигурации
Код загружает настройки из файлов config.json и .env (например, порт сервера, параметры базы данных, настройки логирования). Это делается с помощью функции config.LoadConfig().
Инициализация логирования
Создаётся логгер, который записывает информацию о работе приложения (сообщения, ошибки) в файл, указанный в конфигурации.
Подключение к базе данных
Устанавливается соединение с базой данных PostgreSQL с использованием параметров из конфигурации. Если соединение не удаётся, программа завершает работу с ошибкой.
Инициализация базы данных
Выполняются миграции (создание таблиц и других структур в базе данных), чтобы подготовить её к работе.
Создание сервисов и контроллеров
Создаются экземпляры сервисов (например, для аутентификации и управления курсами) и контроллеров, которые обрабатывают бизнес-логику и HTTP-запросы.
Настройка маршрутов
Используется фреймворк Gin для определения маршрутов API (например, /register, /login, /api/courses), которые связываются с методами контроллеров.
Запуск сервера
Сервер запускается на указанном в конфигурации порту и начинает обрабатывать входящие запросы.
Этот файл — "сердце" приложения, которое связывает все модули вместе и обеспечивает его работу.
